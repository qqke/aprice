# Lessons

- Auth 页面和顶栏必须一起跟随 session state 切换，不能只改登录调用链而保留静态“登录”入口。
- 用户可见文案里不要直接暴露技术供应商标签；如果必须出现，只能放在 meta 或注释里。
- 已登录返回登录页时，应该优先展示会话摘要和切换/退出动作，隐藏注册、登录、忘记密码表单。
- 正式页面里尽量统一使用“账号 / 登录状态 / 个人页 / 管理员权限”这类产品化措辞，避免“当前会话 / profile / 去我的”之类的调试或口语表达。


- 用户可见状态文案要尽量短，优先用结果句而不是说明句；避免同时出现‘这里/会显示/会自动/先’这类引导词。

- 全站按钮动作词要统一，同一类动作尽量只用一种说法，例如只保留‘打开个人页’、‘按当前位置排序’、‘最新添加’。

- 页面里复用同一套趋势/标签逻辑时，要么同步把 helper 放到每个引用页，要么下沉到公共模块；不能只删一处实现。


- 收尾时优先做一次全局文案清理，把‘先 / 正在 / 暂无 / 会显示 / 会自动’集中收短，避免页面像说明书。

- 公开的 Supabase URL / anon key 也不要写成源码默认值；GitHub Actions 只读 Secrets，前端和同步脚本都应该在缺失时明确报错。

- 构建/测试前必须先重新生成 `dist`，不要拿旧产物判断页面状态；如果 `dist` 反复与源码不同步，就先检查预构建脚本和测试流程是否都在跑新产物。

- 构建和测试不能并行跑；凡是依赖 `dist` 的浏览器测试，都必须先等 `npm run build` 完成，再开始 `npm test`，否则会读到半成品或旧产物。

- 依赖 `dist` 的浏览器测试和 `npm run build` 不能并行；必须先完整构建，再开始测试，否则容易读到半成品或旧产物。

- 顶部会话状态的过渡文案不要用“同步中/刷新中”，切页时应尽量直接切到已登录基准态；如果必须异步刷新，也要先用稳定外观占位。

- 顶栏切页去闪要优先在 `head` 里提前打本地会话标记，再用 CSS 隐藏中间态；不要把“已登录/同步中”这类过渡写在 body 里。

- 登录页如果依赖 `session-hint` 判断已登录，首屏就要同时隐藏登录表单和会话概览，不能等 auth 模块加载后再切状态；否则会先闪出登录框。
- Astro 的静态 404 只负责产物里的 `404.html`；开发态要想让未知路径也显示自定义页面，应该用 middleware 把 404 重写到 `/404`，不要直接加静态 catch-all 路由。
